# 애널라이저

구성요소

1. 캐릭터 필터 (0개 이상)
2. 토크나이저 (1개)
3. 토큰 필터 (0개 이상)

```bash
POST _analyze

{
    "analyzer": "standard", # 사용할 애널라이저
    "text": [애널라이저를 적용시킬 문자열]
}
```

## 캐릭터 필터

텍스트를 캐릭터의 스트림으로 받아 특정한 문자를 추가, 변경, 삭제하는 필터

```bash
POST _analyzer

{
    "char_filter": [
        "html_strip",
        {
            "type": "mapping",
            "mappings": [
                "바보 => 천재",
                ":) => _happy_"
            ]
        },
        {
            "type": "pattern_replace",
            "pattern": "(\\d+)-(?=\\d)",
            "replacement": "$1_"
        }
    ],
    "text": [애널라이저를 적용시킬 문자열]
}
```

- html_strip → html 요소 데이터를 꺼내고, html 엔티티 디코딩
- mapping → 치환할 대상이 되는 문자와 치환 문자를 맵형태로 선언
- pattern_replace → 정규 표현식을 이용해 문서 치환

## 토크나이저

캐릭터 스트림을 받아 여러 토큰으로 쪼개어 토큰 스트림을 만듦

### standard 토크나이저

unicode text segmentation 알고리즘을 사용해 텍스트를 단어 단위로 나눔

대부분의 문장 부호가 사라지고 standard 애널라이저는 해당 토크나이저를 사용

### keyword 토크나이저

텍스트를 쪼개지 않고 단일 토큰으로 내보냄

### ngram 토크나이저

텍스트를 min_gram ~ max_gram 단위로 쪼갬 (둘 사이의 차이는 1)

hello → he, hel, el, ell, ll, loo (min_gram: 2, max_gram: 3)

token_chars 속성 → 토큰에 포함시킬 타입 문자를 지정할 수 있음

- Letter → 언어의 글자로 분류되는 문자
- Digit → 숫자로 분류되는 문자
- whitespace → 공백으로 인식되는 문자 (줄바꿈 문자, 띄어쓰기)
- punctuation → 문장부호
- symbol → $, ≥ 같은 기호
- custom → custom_token_chars 설정을 통해 따로 지정한 커스텀 문자

```bash
POST _analyzer

{
    "tokenizer": {
        "type": "ngram",
        "min_gram": 3,
        "max_gram": 4,
        "token_chars": ["letter"]
    },
    "text": [애널라이저를 적용시킬 문자열]
}
```

### edge_ngram 토크나이저

ngram과 유사하게 동작하지만, 모든 토큰의 시작 글자를 단어의 시작 글자로 고정시켜서 생성

```bash
POST _analyzer

{
    "tokenizer": {
        "type": "edge_ngram",
        "min_gram": 3,
        "max_gram": 4,
        "token_chars": ["letter"]
    },
    "text": [애널라이저를 적용시킬 문자열]
}
```

### 그 외 토크나이저

- letter → 글자로 분류되는 문자가 아닌 문자(공백,  특수문자 등)를 만났을 때 쪼갬
- whitespace → 공백을 만났을 때 쪼갬
- pattern → 지정한 정규표현식을 단어의 구분자로 사용하여 쪼갬

## 토큰 필터

토큰 스트림을 받아 토큰을 추가, 변경, 삭제

여러 개의 토큰 필터가 지정된 경우 순차 적용

**내장 토큰 필터**

- **lowercase / uppercase** → 토큰의 내용을 소/대문자롤 변경
- **stop** → 불용어를 지정하여 제거 가능 (the, a, an, in …)
- **synonym** → 유의어 사전 파일을 지정하여 지정된 유의어 치환
- **pattern_replace** → 정규식을 사용하여 토큰 내용 치환
- **stemmer** → 어간 추출 수행
- **trim** → 토큰 전후에 위치한 공백 문자 제거
- **truncate** → 지정한 길이로 토큰을 ㅈ

### 내장 애널라이저

- **standard** → standard 토크나이저 + lowercase 토큰 필터 (기본)
- **simple** → 문자 단위로 토큰 스트림 생성 + lowercase 토큰 필터
- **whitespace** → 공백 단위로 토큰 생성
- **stop** → standard랑 동일하게 동작 + stop 토큰 필터로 불용어 제거
- **keyword** → 하나의 큰 토큰을 그대로 반환
- **pattern** → pattern 토크나이저 + lowercase 토큰 필터
- **language** → 여러 언어의 분석 지원 (한국어 미지원)
- **fingerprint** → 동격 문자 치환, 토큰 정렬 후 단일 토큰으로 합침

### 커스텀 애널라이저 생성

```bash
PUT [인덱스 이름]

{
    "settings": {
        "analysis": {
            "char_filter": {
                "my_char_filter": {
                "type": "mapping:",
                "mappings": [
                    "i. => 1.",
                    "ii. => 2.",
                    "iii. => 3.",
                    "iv. => 4."
                ]
            }
        }
    },
    "analyzer": {
        "my_analyzer": {
            "char_filter": ["my_char_filter"],
            "tokenizer": "whitespace",
            "token_filter": "lowercase"
        }
    },
    "mappings": {
        "properties": {
            "test_text": {
            "type": "text",
            "analyzer": "my_analyzer"
            }
        }
    }
}
```

## 노멀라이저

하나의 단일 토큰을 처리하기 위한 캐릭터 필터와 토큰 필터로 이루어짐

애널라이저와 달리 토크나이저를 포함하지 않으며, keyword 타입 필드에만 적용 가능

keyword 필드에서 대소문자 구분 없이 정확히 일치하는 검색, 집계(aggregation) 시 값 정규화 등에 사용

```bash
PUT my_index

{
    "settings": {
        "analysis": {
            "normalizer": {
                "my_normalizer": {
                    "type": "custom",
                    "char_filter": [],
                    "filter": ["lowercase", "asciifolding"]
                }
            }
        }
    },
    "mappings": {
        "properties": {
            "tag": {
                "type": "keyword",
                "normalizer": "my_normalizer"
            }
        }
    }
}
```

“HELLO”, “Hello”, “hello”는 모두 “hello”로 정규화되어 동일한 값으로 처리됨 (lowercase 필터 적용)

asciifolding → é, ü, ñ 같은 유니코드 문자를 ASCII 동등 문자(e, u, n)로 변환하여 다국어 정렬/검색에 활용

## 인덱스 애널라이저 vs 검색 애널라이저

Elasticsearch는 인덱싱 시점과 검색 시점에 서로 다른 애널라이저를 적용할 수 있음

**analyzer** → 인덱싱과 검색 모두에 적용되는 기본 애널라이저

**search_analyzer** → 검색 시점에만 적용되는 애널라이저로, analyzer와 다르게 설정 가능

대표적인 패턴: edge_ngram을 인덱스 애널라이저로 사용하고, standard를 검색 애널라이저로 사용하면 자동완성(접두어 검색) 구현 가능

```bash
PUT [인덱스 이름]

{
    "settings": {
        "analysis": {
            "analyzer": {
                "autocomplete_index_analyzer": {
                    "tokenizer": "autocomplete_tokenizer",
                    "filter": ["lowercase"]
                }
            }
        }
    }
}
```

**elastic**을 인덱싱하면 `el, ela, elas, … elastic`의 토큰이 생성

사용자가 `ela`로 검색하면 search_analyzer는 `ela` 하나의 토큰만 생성 → 인덱스에 있는 ela 토큰과 매칭되어 검색 결과 반환
